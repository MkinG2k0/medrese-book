# Phase 9: Realtime-уведомления и Web Push — контекст

**Создано:** 2026-06-25  
**Статус:** Ожидает планирования (`/gsd-plan-phase 9`)

## Цель

Расширить in-app уведомления (Phase 6) до **realtime-доставки** и **Web Push** с VAPID-ключами. Сообщения между пользователями (чат) — отдельно, Phase 7; здесь только **событийные push-уведомления**.

## Scope

1. **Realtime-канал** — доставка новых уведомлений без перезагрузки страницы (SSE, WebSocket или polling — выбрать при планировании).
2. **Web Push API** — подписка браузера, VAPID-ключи в env, service worker.
3. **Колокольчик для всех ролей** — единая система уведомлений (база из Phase 6).
4. Первые события для push:
   - новая заявка на отпуск/отгул (менеджеру)
   - решение по заявке (преподавателю)
   - назначение замещения (замещающему)

## Вне scope (позже)

- Личные сообщения / чат — Phase 7
- Email / SMS / Telegram

## Зависимости

- **Phase 6** — модель Notification, колокольчик, in-app список
- **Phase 8** — доменные события заявок и замещений как источник push

## Env / инфра

- `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` (mailto или URL)
- Service worker в `public/` или Next.js PWA-паттерн
