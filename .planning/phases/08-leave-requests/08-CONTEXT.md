# Phase 8: Отпуска, отгулы и больничные — контекст

**Создано:** 2026-06-25  
**Статус:** Ожидает планирования (`/gsd-plan-phase 8`)

## Связь с Phase 4

Phase 4 (Substitution & Leave) задаёт базовые требования TCHR-01…05 и LEAV-01…04.  
Phase 8 детализирует **полный UX-пайплайн** заявок на отсутствие, включая **больничные**, календари, согласование и привязку замещения. При планировании Phase 8 — уточнить границы с Phase 4 и избежать дублирования.

## Пайплайн (учитель)

1. **Страница преподавателя «Календарь»** (новая или расширение существующей).
2. Кнопки: **«Создать отпуск»**, **«Создать отгул»** (больничный — отдельный тип или подтип отгула, уточнить при планировании).
3. По клику — **модалка**:
   - Range DatePicker (диапазон дат)
   - Поле **описание** (причина / комментарий)
4. После отправки создаётся **заявка** со статусом **`СОЗДАНА`**.

## Пайплайн (менеджер)

1. **Уведомление** о новой заявке — через in-app колокольчик (Phase 6). Realtime / Web Push — Phase 9.
2. **Страница менеджера «Календарь отпусков»**:
   - Календарь **всех** заявок преподавателей
   - **`СОЗДАНА`** — серый цвет на календаре
   - **`ПОДТВЕРЖДЕНА`** — зелёный
   - **`ОТМЕНЕНА` / отклонена** — **не отображается** на календаре
3. **Ниже календаря** — грид всех заявок (фильтры, статусы, преподаватель, даты).
4. Менеджер открывает заявку и **принимает** или **отклоняет**:
   - При **отклонении** — обязательное поле **причина** (например: «этот день занят»).
   - При **принятии** — обязательный выбор **замещающего преподавателя** на период отсутствия.

## Пайплайн (замещающий преподаватель)

1. In-app **уведомление**: «Вы замещаете {ФИО преподавателя} с {дата} по {дата}».
2. Замещающий может **перейти в учётку замещаемого** через селект «Сменить учётку» в левом меню.
3. **Ограничение:** преподаватели переключают учётки **только** при активном официальном замещении (TCHR-01). Свободное переключение между преподавателями запрещено.

## Статусы заявки

| Статус | Календарь менеджера | Действия |
|--------|---------------------|----------|
| СОЗДАНА | Серый | Ожидает решения менеджера |
| ПОДТВЕРЖДЕНА | Зелёный | Замещение активировано |
| ОТМЕНЕНА / ОТКЛОНЕНА | Скрыта | Причина отклонения сохранена |

## Зависимости

- **Phase 2** — управление преподавателями, карточки
- **Phase 6** — колокольчик и in-app уведомления (новая заявка, решение, назначение замещения)
- **Phase 4** — базовая модель замещения (TCHR-03…05); Phase 8 связывает замещение с одобрением заявки

## Вне scope Phase 8

- Realtime-доставка и **Web Push API (VAPID)** — **Phase 9**
- Чат и сообщения между пользователями — **Phase 7**
