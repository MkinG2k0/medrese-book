# Feature Landscape

**Domain:** Электронный дневник медресе (журнал посещаемости и успеваемости по программе Корана)
**Researched:** 2026-06-24
**Confidence:** HIGH для table stakes (электронный журнал, Дневник.ру, МЭШ); MEDIUM для enterprise LMS (Canvas, D2L); LOW для предиктивной аналитики и ML-фич

## Контекст medrese-book

Продукт — **узкоспециализированный журнал**, не полноценный LMS. Ученик привязан к одному преподавателю и группе; прогресс — по **шагам программы** (уровни/шаги), а не по предметам. Роли: SUPER_ADMIN, MANAGER, TEACHER, STUDENT (роли «родитель» нет). Уже реализованы: журнал, базовая месячная аналитика (топ учеников, статистика по уровням), портал ученика, админка программы.

Исследование сопоставляет экосистему электронных журналов (Дневник.ру, МЭШ, Импульс) и LMS (Canvas, D2L, PowerSchool) с **утверждённым бэклогом** medrese-book.

## 1. Аналитика обучения ученика

### Table Stakes

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| История занятий с датой, посещаемостью, оценками | Базовая функция любого электронного дневника; без неё журнал бесполезен | Low | Уже частично есть (Session + StepCompletion); нужна единая «лента истории» |
| Прогресс по уровням и шагам | Ученик и учитель должны видеть «где мы сейчас» | Low | Портал ученика + `currentStepIdx` — есть; нужна детализация по шагам |
| Средний балл за период | Стандартный отчёт (Дневник.ру, МЭШ) | Low | Есть в месячной аналитике; нужен на карточке ученика |
| Посещаемость: пропуски, опоздания | Обязательный учёт в регламентах электронного журнала (РФ 2025) | Low | Есть в Session; нужна агрегация в истории |
| Фильтр аналитики по периоду (месяц/четверть) | Все журналы строят отчёты «за период» | Low | Месячный picker есть; расширить на карточку |
| Количество пройденных шагов/уроков | Минимальная метрика прогресса для программного обучения | Low | В бэклоге фазы 1 |
| Исключение «зачтено ранее» из метрик | Иначе аналитика врёт при переводе ученика | Med | Техдолг: `syncCompletionsForProgress` создаёт синтетические сессии |
| Предупреждение при отставании от норматива | Для medrese — **доменный** норматив 48 ч на 1-й уровень; в массовых журналах — пороги по СОУ/качеству | Med | Дифференциатор medrese, но **обязателен по бэклогу** |

### Differentiators

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| Длительность каждого урока (ручной старт/завершение) | Точное время обучения для норматива 48 ч; в массовых журналах обычно только «урок состоялся» | Med | Редко в школьных журналах; ценно для индивидуального обучения Корану |
| Общее время обучения (сумма длительностей) | KPI для менеджера: «сколько часов вложено» | Med | Зависит от таймера урока |
| Таймер текущего урока в журнале | UX для учителя; снижает забывание завершить урок | Low | В бэклоге фазы 1 |
| Единая хронологическая лента (занятия + оценки + допзадания) | Сводный timeline — удобнее разрозненных экранов | Med | Фаза 1–2; допзадания подключаются позже |
| Аналитика преподавателя (время входа vs начало урока) | Контроль дисциплины преподавания | Med | Фаза 3; не table stakes для ученика |
| Сравнение ученика с когортой (топ, среднее по уровню) | Менеджер видит отстающих | Low | TopStudents + LevelStats — частично есть |
| Экспорт отчётов (CSV/PDF) | Ожидают администраторы в крупных системах | Med | Нет в бэклоге v1 — отложить |
| Предиктивная аналитика «ученик в зоне риска» | Canvas/D2L продвигают at-risk scoring | High | **Anti-feature для medrese v1** — мало данных, малый масштаб |

### Anti-Features

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| ML/AI-предсказание отсева | Переинженеринг для десятков учеников; низкая достоверность | Простые пороги: норматив 48 ч, N пропусков, средний балл < X |
| 95+ фильтров и BI-дашборды (LearnWorlds-уровень) | Нет аудитории аналитиков; раздувает scope | 3–5 ключевых срезов: период, преподаватель, уровень, ученик |
| xAPI/LRS-интеграция | Нет внешних систем потребления | Прямые Prisma-запросы + revalidatePath |
| Real-time дашборды (WebSocket) | Нет операционной нужды | SSR + React Query polling по необходимости |
| Peer comparison для учеников | Духовное обучение — не соревнование | Только для менеджера/учителя, не в портале ученика |

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|

| Аналитика допзаданий (% выполнения, популярность) | Менеджер видит эффективность интервенций | Med | **Отложено** в бэклоге (низкий приоритет) |
| Автоназначение при срабатывании норматива 48 ч | Проактивная интервенция | High | Возможная фаза 4; не в текущем scope |
| Прикрепление файлов к заданию | МЭШ, Дневник.ру — стандарт | Med | Out of scope: локальные uploads только для программы |

### Anti-Features

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| Цифровое ДЗ с автопроверкой (тесты) | Другой продукт; medrese — устное/практическое обучение | Текст + ручная проверка учителем |
| Массовая выдача по классу (30+ учеников) | Модель 1:1/малые группы | Назначение per student |
| GitHub-style submission workflow | Избыточно | Статус + комментарий в журнале |
| Интеграция с внешней «библиотекой» материалов | Нет контента-партнёров | Ссылка на шаг программы |

## 3. Замещение преподавателей и отпуска

### Table Stakes

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| Календарь отсутствий (отпуск/больничный/отгул) | Любая школа фиксирует отсутствие педагога | Med | Фаза 2 |
| Заявка преподавателя + согласование менеджером | Workflow approval — стандарт HR/школьных систем | Med | Tyler, Frontline, OpenEduCat |
| Назначение замещающего на период | Без замены журнал недоступен | Med | Проще чем в K-12 (нет расписания по кабинетам) |
| Ограничение срока замещения | Предотвращает бессрочный доступ чужого учителя | Low | В бэклоге |
| Журнал замещений (кто, кого, когда) | Аудит и справедливое распределение нагрузки | Low | Table stakes для compliance |
| Авто-замещение при одобренном отпуске | Снижает ручную работу менеджера | Med | В бэклоге; можно начать с ручного назначения |
| Запрет входа под чужой учёткой (кроме официального замещения) | Безопасность; сейчас `switchUser` — уязвимость | Med | Фаза 2 безопасности |

### Differentiators

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| Смена постоянного преподавателя с обязательной причиной | Для medrese — частая операция (перевод между учителями) | Med | Отдельно от краткосрочного замещения |
| Уведомление замещающему (push/SMS) | K-12 системы — за минуты; для medrese email/in-app достаточно | Low | Через фазу 3 notifications |
| Статистика нагрузки замещений (fairness) | K-12: weekly caps, round-robin | Med | Полезно при >5 преподавателях |
| Интеграция с расписанием/таймтейблом | K-12 table stakes | High | **Не нужно** — medrese без почасового расписания |
| Пул внешних заменителей | Крупные школы | High | Anti-feature — все замены внутренние |

### Anti-Features

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| Автоматический matching по предмету/кабинету | Нет предметов и кабинетов | Matching по группе/ученику |
| Компенсации/отгулы за замены | HR/payroll — вне scope | Только учёт фактов замещения |
| Fill-rate оптимизация и прогноз отсутствий | Enterprise K-12 (Frontline) | Простой календарь + ручное назначение |
| Замена через общий код учителя | Текущий антипаттерн | JWT с `substitutingForTeacherId` + audit |

## 5. Аудит и безопасность

### Table Stakes

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| Журнал аудита с фильтрами (субъект, тип, дата) | FERPA §99.32, регламенты журналов — кто что менял | Med | Фаза 3; сейчас **нет** (CONCERNS.md) |
| Логирование входов/выходов | Базовая security hygiene | Low | Фаза 2 |
| Автовыход после бездействия (1 ч) | Session timeout — стандарт | Low | Фаза 2 |
| Аудит смены преподавателя, прогресса, статусов | Immutable trail для спорных ситуаций | Med | Критично для medrese |
| Append-only запись (без редактирования логов) | Tamper-evident trail | Med | Отдельная таблица `AuditLog` |
| Кто / что / когда / до-после (для мутаций) | Минимальный контракт audit event | Med | JSON `payload` для деталей |

### Differentiators

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| Аудит impersonation/switch-user | CONCERNS.md рекомендует | Med | SUPER_ADMIN/MANAGER switch |
| IP-адрес и session ID в логе | FERPA best practice | Low | Опционально |
| Экспорт audit log для проверки | Compliance | Med | Фаза 4 |
| Retention policy (3+ года) | FERPA/корпоративные стандарты | Low | Политика на уровне БД |
| Алерты на аномальный доступ | SIEM-уровень | High | Overkill для medrese |

### Anti-Features

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| Смешивание audit log с `console.error` | Невозможно расследовать | Dedicated `AuditLog` table |
| Логирование каждого GET-запроса | Шум, GDPR/объём | Только мутации и чувствительные чтения |
| Полная FERPA-compliance suite | Medrese в РФ, не US federal funding | Принципы privacy без юридического overfit |
| WORM-хранилище / blockchain audit | Enterprise | PostgreSQL append-only + no UPDATE policy |

## Feature Dependencies

```
Ручной таймер урока (старт/завершение)
  → Длительность урока
    → Общее время обучения
      → Норматив 48 ч + предупреждение

Исключение «зачтено ранее» из аналитики
  → Корректные метрики времени и шагов
    → Норматив 48 ч (иначе ложные срабатывания)

История ученика (фаза 1)
  → Карточка ученика (фаза 2)
    → Допзадания в истории
    → Статусы ученика

Справочник допзаданий
  → Назначение допзадания
    → Уведомление ученику/учителю

Календарь отпусков + approval
  → Замещение с ограничением срока
    → JWT/сессия с контекстом замещения
      → Запрет switchUser для учителей

AuditLog (базовая таблица)
  → Журнал аудита UI
  → Аудит смены преподавателя, допзаданий, отпусков
  → Аналитика преподавателя (время входа vs урок)

Уведомления (колокольчик)
  → Зависит от событий: оценки, допзадания, отпуска, норматив
  → НЕ блокирует фазы 1–2 (можно поставить после доменных сущностей)
```

## MVP Recommendation (для medrese-book)

### Приоритет 1 — Table Stakes ядра (фаза 1)

1. Длительность урока + таймер + общее время
2. Прогресс по уровням/шагам + количество пройденного
3. История ученика (занятия, оценки, время)
4. Исключение «зачтено ранее» из статистики
5. Норматив 48 ч с предупреждением

**Обоснование:** без этого Core Value («реальный прогресс и вовремя вмешаться») не достижим.

### Приоритет 2 — Операционное управление (фаза 2)

1. Карточка ученика + статусы
2. Справочник и назначение допзаданий
3. Смена преподавателя с причиной
4. Отпуска + замещение + запрет чужого входа
5. Логин/логаут + session timeout

### Приоритет 3 — Наблюдаемость (фаза 3)

1. Уведомления (колокольчик, 5–7 типов событий)
2. Журнал аудита с фильтрами
3. Аналитика преподавателя

### Defer

| Feature | Reason |
|---------|--------|
| Аналитика допзаданий | Низкий приоритет в бэклоге; нужны данные после фазы 2 |
| Чат между ролями | Фаза 3, отдельный scope; не блокирует уведомления |
| Экспорт CSV/PDF | Нет запроса; ручной просмотр достаточен на старте |
| Email/SMS/Telegram | Нет роли родителя; in-app first |
| Предиктивная аналитика | Anti-feature для масштаба medrese |
| Оценки звёздочками, новый дизайн уведомлений | Косметика; низкий приоритет |

## Сводка: Table Stakes vs Differentiators для ЭТОГО бэклога

| Область | Table Stakes (must have) | Differentiators (medrese) | Anti-Features |
|---------|--------------------------|---------------------------|---------------|
| **Аналитика** | История, прогресс, посещаемость, средний балл, период | Таймер урока, 48 ч норматив, исключение зачёта | ML risk scoring, xAPI, 95 фильтров |
| **Допзадания** | Назначение, статус, срок, результат, справочник | Привязка к шагу, кастомные типы | Автотесты, массовая выдача классу |
| **Замены/отпуска** | Календарь, approval, замещение, журнал | Смена пост. преподавателя с причиной | Таймтейбл-matching, payroll |
| **Уведомления** | Колокольчик, непрочитанные, события обучения | Пороговые алерты | WebSocket-чат, SMS-first |
| **Аудит** | Журнал мутаций, login/logout, timeout | Impersonation audit | FERPA suite, GET-логирование |
