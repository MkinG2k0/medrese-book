# Requirements: Электронный дневник медресе

**Defined:** 2026-06-24
**Core Value:** Учитель и менеджер видят реальный прогресс каждого ученика и могут вовремя вмешаться

## v1 Requirements

### Foundation (техническая база)

- [ ] **FND-01**: Единый фильтр аналитики исключает сессии-корректировки и шаги «зачтено ранее» из метрик
- [ ] **FND-02**: Общий `authorizeApiRequest()` для всех API routes с default-deny
- [ ] **FND-03**: Централизованный модуль `student-progress` для пересчёта `currentStepIdx` и prior credit
- [ ] **FND-04**: `dispatchDomainEvent` для fan-out в audit и notifications без cross-feature imports

### Аналитика ученика (фаза 1)

- [ ] **ANLY-01**: Преподаватель вручную запускает и завершает урок; система сохраняет `startedAt`/`endedAt` и длительность
- [ ] **ANLY-02**: В журнале отображается таймер текущего урока (elapsed time)
- [ ] **ANLY-03**: Ученик/учитель/менеджер видят количество пройденных уроков за период
- [ ] **ANLY-04**: Ученик/учитель/менеджер видят количество пройденных шагов за период
- [ ] **ANLY-05**: Ученик/учитель/менеджер видят общее время обучения (сумма длительностей уроков)
- [ ] **ANLY-06**: Отображается прогресс по уровням и шагам программы
- [ ] **ANLY-07**: При превышении норматива 48 часов на 1-м уровне показывается предупреждение
- [ ] **ANLY-08**: История ученика: хронологическая лента занятий, оценок и шагов
- [ ] **ANLY-09**: История ученика включает время обучения по каждому занятию
- [ ] **ANLY-10**: Шаги «зачтено ранее» не учитываются в аналитике и статистике

### Управление учениками (фаза 2)

- [ ] **STUD-01**: Менеджер создаёт ученика с ФИО, телефонами (ученик/опекун), уровнем, шагом и преподавателем
- [ ] **STUD-02**: Карточка ученика: основные данные, контакты опекуна, текущий преподаватель
- [ ] **STUD-03**: Карточка ученика: история обучения, успеваемость, дополнительные задания
- [ ] **STUD-04**: Менеджер меняет преподавателя ученика с обязательной причиной
- [ ] **STUD-05**: Менеджер переводит ученика между уровнями с фиксацией в истории
- [ ] **STUD-06**: Статусы ученика: активный, на паузе, завершил обучение, архивный (вместо удаления)
- [ ] **STUD-07**: При смене статуса обязательна причина (завершил, отказался, переведён, другое)

### Дополнительные задания (фаза 2)

- [ ] **ASGN-01**: Справочник заданий: фиксированные и пользовательские типы
- [ ] **ASGN-02**: При средней оценке ниже порога учитель видит кнопку «Назначить дополнительное задание»
- [ ] **ASGN-03**: Модальное окно со списком заданий для текущего шага; можно назначить несколько
- [ ] **ASGN-04**: Учитель/менеджер создаёт собственное задание в справочнике
- [ ] **ASGN-05**: Отображается автор задания
- [ ] **ASGN-06**: Страница всех заданий с фильтрацией по шагам и уровням
- [ ] **ASGN-07**: Назначенные задания отображаются в истории ученика

### Преподаватели и замещение (фаза 2)

- [ ] **TCHR-01**: Запрещён вход под учёткой другого преподавателя без официального замещения
- [ ] **TCHR-02**: Менеджер видит коды преподавателей
- [ ] **TCHR-03**: Менеджер назначает замещающего преподавателя с ограничением по сроку
- [ ] **TCHR-04**: Замещающий получает доступ к ученикам замещаемого на период замещения
- [ ] **TCHR-05**: Ведётся история замещений

### Отпуска и отгулы (фаза 2)

- [ ] **LEAV-01**: Календарь отпусков и отгулов преподавателей
- [ ] **LEAV-02**: Преподаватель создаёт заявку на отпуск или отгул
- [ ] **LEAV-03**: Менеджер согласовывает или отклоняет заявку (при отклонении — обязательная причина)
- [ ] **LEAV-04**: При одобренном отпуске автоматически активируется назначенное замещение

### Безопасность (фаза 2)

- [ ] **SECU-01**: Автоматический выход после 1 часа бездействия
- [ ] **SECU-02**: Логирование входов и выходов пользователей

### Уведомления (фаза 3)

- [ ] **NOTF-01**: Колокольчик с количеством непрочитанных уведомлений
- [ ] **NOTF-02**: Уведомления о непрочитанных событиях (in-app)
- [ ] **NOTF-03**: Уведомления об успеваемости (норматив, низкая оценка)
- [ ] **NOTF-04**: Системные уведомления (согласование отпуска, замещение)

### Аудит (фаза 3)

- [ ] **AUDT-01**: Журнал действий: пользователь, дата/время, тип, старое/новое значение
- [ ] **AUDT-02**: История ученика в аудите: создание, изменение данных, смена преподавателя, уровня, шага, оценки, задания, статуса
- [ ] **AUDT-03**: История преподавателя в аудите: входы, начало/окончание занятий, замещения, отпуска
- [ ] **AUDT-04**: Фильтрация журнала по дате, пользователю, ученику, преподавателю, типу события

### Аналитика преподавателя (фаза 3)

- [ ] **TANL-01**: Фиксируется время входа преподавателя в систему
- [ ] **TANL-02**: Фиксируется время начала урока через журнал
- [ ] **TANL-03**: Отображается разница между входом и началом урока
- [ ] **TANL-04**: Статистика проведённых уроков по преподавателю

### Коммуникации — чат (фаза 3)

- [ ] **CHAT-01**: Личные диалоги между ролями (кроме супер-администратора)
- [ ] **CHAT-02**: История сообщений в диалоге
- [ ] **CHAT-03**: Менеджер видит все чаты системы и раздел «Мои чаты»

## v2 Requirements

### Аналитика допзаданий

- **AASN-02**: Процент выполнения допзаданий
- **AASN-03**: Средняя оценка после выполнения
- **AASN-04**: Самые популярные задания; аналитика по шагам и уровням

### Интерфейс

- **UI-02**: Обновлённый дизайн уведомлений
- **UI-03**: Замена иконки приложения

### Экспорт и интеграции

- **EXPT-01**: Экспорт отчётов в CSV/PDF
- **EXPT-02**: Email/SMS/Telegram-уведомления

## Out of Scope

| Feature | Reason |
|---------|--------|
| Мобильное нативное приложение | Веб достаточен для медресе |
| ML/предиктивная аналитика | Малый масштаб, низкая достоверность |
| xAPI/LRS | Нет внешних потребителей данных |
| Real-time WebSocket-чат в MVP | Асинхронная модель при внедрении чата |
| S3-хранилище | Локальные uploads достаточны |
| Почасовое расписание/кабинеты | Модель medrese — индивидуальное обучение |
| Автопроверка ДЗ (тесты) | Устное/практическое обучение Корану |
| Полная переработка дизайна | Точечные улучшения по бэклогу |
