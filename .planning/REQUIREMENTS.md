# Requirements: Электронный дневник медресе

**Defined:** 2026-06-24
**Core Value:** Учитель и менеджер видят реальный прогресс каждого ученика и могут вовремя вмешаться

## v2.0 Requirements — Система предметов

**Milestone:** v2.0 Система предметов
**Defined:** 2026-07-07

### Предметы и программа

- [x] **SUBJ-01**: Менеджер и супер-админ создают, редактируют и удаляют предметы
- [x] **SUBJ-02**: У каждого предмета своя учебная программа: уровни → шаги (структура как текущая глобальная программа)
- [x] **SUBJ-03**: Программа — шаблон на предмет: все группы по одному предмету используют одну программу
- [x] **SUBJ-04**: Редактор программы предмета (уровни/шаги, Tiptap) доступен менеджеру и супер-админу

### Группы и зачисление

- [x] **SUBJ-05**: Группа привязана ровно к одному предмету
- [x] **SUBJ-06**: Ученик может быть зачислен в несколько групп (несколько предметов одновременно)
- [x] **SUBJ-07**: Менеджер назначает предмет при создании/редактировании группы

### Прогресс и сессии

- [x] **SUBJ-08**: Прогресс ученика хранится независимо по каждому зачислению в группу (уровень + текущий шаг на GroupEnrollment); ученик в двух группах одного предмета имеет два прогресса
- [x] **SUBJ-09**: Сессия уникальна по ученику + дате + groupId; посещаемость и оценки в рамках занятия группы
- [x] **SUBJ-10**: Пересчёт `currentStepIdx` и prior credit работает в скоупе зачисления (groupId/enrollment); auto-promote только текущего enrollment

### Журнал

- [x] **SUBJ-11**: Учитель выбирает группу в журнале; предмет определяется автоматически из группы
- [x] **SUBJ-12**: Список учеников группы показывает прогресс по предмету этой группы
- [x] **SUBJ-13**: Урок (оценки по шагам, посещаемость) ведётся в контексте предмета группы

### Аналитика

- [ ] **SUBJ-14**: На странице аналитики менеджер/учитель выбирает предмет
- [ ] **SUBJ-15**: Метрики (топ учеников, статистика по уровням, at-risk) считаются в скоупе выбранного предмета

### Портал ученика и допзадания

- [ ] **SUBJ-16**: Ученик видит прогресс по каждому своему предмету
- [ ] **SUBJ-17**: Дополнительные задания привязаны к шагам программы предмета

### Данные

- [x] **SUBJ-18**: Новая Prisma-схема и seed для мультипредметной модели (fresh start, без миграции старых completions)

## v1 Requirements (archived — milestone v1.0)

### Foundation (техническая база)

- [x] **FND-01**: Единый фильтр аналитики исключает сессии-корректировки и шаги «зачтено ранее» из метрик
- [x] **FND-02**: Общий `authorizeApiRequest()` для всех API routes с default-deny
- [ ] **FND-03**: Централизованный модуль `student-progress` для пересчёта `currentStepIdx` и prior credit
- [ ] **FND-04**: `dispatchDomainEvent` для fan-out в audit и notifications без cross-feature imports

### Аналитика ученика (фаза 1)

- [x] **ANLY-01**: Преподаватель вручную запускает и завершает урок; система сохраняет `startedAt`/`endedAt` и длительность
- [x] **ANLY-02**: В журнале отображается таймер текущего урока (elapsed time)
- [x] **ANLY-03**: Ученик/учитель/менеджер видят количество пройденных уроков за период
- [x] **ANLY-04**: Ученик/учитель/менеджер видят количество пройденных шагов за период
- [x] **ANLY-05**: Ученик/учитель/менеджер видят общее время обучения (сумма длительностей уроков)
- [x] **ANLY-06**: Отображается прогресс по уровням и шагам программы
- [x] **ANLY-07**: При превышении норматива 48 часов на 1-м уровне показывается предупреждение
- [x] **ANLY-08**: История ученика: хронологическая лента занятий, оценок и шагов
- [x] **ANLY-09**: История ученика включает время обучения по каждому занятию
- [x] **ANLY-10**: Шаги «зачтено ранее» не учитываются в аналитике и статистике

### Управление учениками (фаза 2)

- [ ] **STUD-01**: Менеджер создаёт ученика с ФИО, телефонами (ученик/опекун), уровнем, шагом и преподавателем
- [ ] **STUD-02**: Карточка ученика: основные данные, контакты опекуна, текущий преподаватель
- [ ] **STUD-03**: Карточка ученика: история обучения, успеваемость, дополнительные задания
- [ ] **STUD-04**: Менеджер меняет преподавателя ученика с обязательной причиной
- [ ] **STUD-05**: Менеджер переводит ученика между уровнями с фиксацией в истории
- [ ] **STUD-06**: Статусы ученика: активный, на паузе, завершил обучение, архивный (вместо удаления)
- [ ] **STUD-07**: При смене статуса обязательна причина (завершил, отказался, переведён, другое)

### Дополнительные задания (фаза 2)

- [ ] **ASGN-01**: Справочник заданий: фиксированные и пользовательские типы
- [ ] **ASGN-02**: При средней оценке ниже порога учитель видит кнопку «Назначить дополнительное задание»
- [ ] **ASGN-03**: Модальное окно со списком заданий для текущего шага; можно назначить несколько
- [ ] **ASGN-04**: Учитель/менеджер создаёт собственное задание в справочнике
- [ ] **ASGN-05**: Отображается автор задания
- [ ] **ASGN-06**: Страница всех заданий с фильтрацией по шагам и уровням
- [ ] **ASGN-07**: Назначенные задания отображаются в истории ученика

### Преподаватели и замещение (фаза 2)

- [x] **TCHR-01**: Запрещён вход под учёткой другого преподавателя без официального замещения
- [ ] **TCHR-02**: Менеджер видит коды преподавателей
- [x] **TCHR-03**: Менеджер назначает замещающего преподавателя с ограничением по сроку
- [x] **TCHR-04**: Замещающий получает доступ к ученикам замещаемого на период замещения
- [x] **TCHR-05**: Ведётся история замещений

### Отпуска и отгулы (фаза 2)

- [x] **LEAV-01**: Календарь отпусков и отгулов преподавателей
- [x] **LEAV-02**: Преподаватель создаёт заявку на отпуск или отгул
- [x] **LEAV-03**: Менеджер согласовывает или отклоняет заявку (при отклонении — обязательная причина)
- [x] **LEAV-04**: При одобренном отпуске автоматически активируется назначенное замещение

### Безопасность (фаза 2)

- [ ] **SECU-01**: Автоматический выход после 1 часа бездействия
- [ ] **SECU-02**: Логирование входов и выходов пользователей

### Уведомления (фаза 3)

- [x] **NOTF-01**: Колокольчик с количеством непрочитанных уведомлений
- [x] **NOTF-02**: Уведомления о непрочитанных событиях (in-app)
- [ ] **NOTF-03**: Уведомления об успеваемости (норматив, низкая оценка)
- [x] **NOTF-04**: Системные уведомления (согласование отпуска, замещение)
- [x] **NOTF-05**: Realtime-доставка in-app уведомлений без перезагрузки (SSE)
- [x] **NOTF-06**: Web Push API с VAPID-ключами и service worker

### Аудит (фаза 3)

- [ ] **AUDT-01**: Журнал действий: пользователь, дата/время, тип, старое/новое значение
- [ ] **AUDT-02**: История ученика в аудите: создание, изменение данных, смена преподавателя, уровня, шага, оценки, задания, статуса
- [ ] **AUDT-03**: История преподавателя в аудите: входы, начало/окончание занятий, замещения, отпуска
- [ ] **AUDT-04**: Фильтрация журнала по дате, пользователю, ученику, преподавателю, типу события

### Аналитика преподавателя (фаза 3)

- [ ] **TANL-01**: Фиксируется время входа преподавателя в систему
- [ ] **TANL-02**: Фиксируется время начала урока через журнал
- [ ] **TANL-03**: Отображается разница между входом и началом урока
- [ ] **TANL-04**: Статистика проведённых уроков по преподавателю

### Коммуникации — чат (фаза 3)

- [ ] **CHAT-01**: Личные диалоги между ролями (кроме супер-администратора)
- [ ] **CHAT-02**: История сообщений в диалоге
- [ ] **CHAT-03**: Менеджер видит все чаты системы и раздел «Мои чаты»

## Out of Scope

| Feature | Reason |
|---------|--------|
| Мобильное нативное приложение | Веб достаточен для медресе |
| ML/предиктивная аналитика | Малый масштаб, низкая достоверность |
| xAPI/LRS | Нет внешних потребителей данных |
| Real-time WebSocket-чат в MVP | Асинхронная модель при внедрении чата |
| S3-хранилище | Локальные uploads достаточны |
| Почасовое расписание/кабинеты | Модель medrese — индивидуальное обучение |
| Автопроверка ДЗ (тесты) | Устное/практическое обучение |
| Полная переработка дизайна | Точечные улучшения по бэклогу |
| Миграция старых completions в v2.0 | Fresh start — новая схема с нуля |
| Копия программы на каждую группу | Программа — шаблон на предмет |
