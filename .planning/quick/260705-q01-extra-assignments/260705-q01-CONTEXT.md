# Quick Task 260705-q01: Доп. уроки и доп. задания - Context

**Gathered:** 2026-07-05
**Status:** Ready for planning

<domain>
## Task Boundary

Функционал дополнительных заданий (доп. уроков):

1. **Отдельная страница** справочника доп. заданий
2. **CRUD**: создавать, редактировать, удалять (только свои), просматривать все (включая чужие)
3. **Автор** виден у каждого задания
4. **Привязка к шагам** программы в справочнике; можно привязать задание с другого шага
5. **На уроке**: после кнопок оценки (1/3/5) — кнопка «Дать доп. задание» (доступна всегда, даже без оценки)
6. **Модалка** выбора заданий с фильтрами: учитель, шаг, название
7. **Назначение** конкретному ученику на текущем уроке; несколько заданий на один шаг
8. **Отображение** назначенных заданий под карточкой шага
9. **Оценивание** доп. заданий теми же кнопками (Средне/Хорошо/Отлично)

Доступ: учителя, менеджеры, супер-админ.

</domain>

<decisions>
## Implementation Decisions

### Содержимое задания
- Rich-контент как у шагов программы — блоки (текст, арабский, картинки) через Tiptap / StepContent

### Кнопка назначения
- Показывать всегда после блока оценки, даже если оценка не выставлена

### Модель назначения
- Справочник (шаблоны) + на уроке учитель выбирает и назначает конкретному ученику
- Задание появляется под карточкой шага только у этого ученика на этом уроке

### Множественность
- Несколько доп. заданий на один шаг одному ученику

### Доступ
- Учителя, менеджеры, супер-админ

### Claude's Discretion
- Структура Prisma-моделей (ExtraAssignment template + StudentExtraAssignment instance + ExtraAssignmentCompletion)
- Переиспользование StepContentPreview / BlockRenderer для отображения
- URL страницы: `/extra-assignments` или `/assignments`
- E2E тесты по правилу new-module-tests

</decisions>

<specifics>
## Specific Ideas

- Фильтры в модалке: по учителям (авторам), по шагам, по названию
- Можно назначить задание с другого шага (cross-step)
- Удалять можно только свои задания в справочнике

</specifics>
