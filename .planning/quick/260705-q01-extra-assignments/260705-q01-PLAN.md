## Task 1 (Wave 1): Data layer — Prisma, validations, API, entities

**Files:**
- `prisma/schema.prisma`
- `prisma/migrations/*/migration.sql`
- `src/shared/lib/validations/extra-assignment.ts`

- `src/app/api/extra-assignments/[id]/route.ts`
- `src/app/api/extra-assignments/assign/route.ts`
- `src/app/api/extra-assignments/instances/[id]/grade/route.ts`

**Action:**

1. **Prisma** — три модели (discretion из CONTEXT):
   - `ExtraAssignment` — шаблон справочника: `title`, `content` (Json, StepContent), `stepId` (nullable FK → Step, для привязки/фильтра), `authorId` (FK → User), `isSystem` (Boolean, для фиксированных ASGN-01), timestamps
   - `StudentExtraAssignment` — экземпляр на уроке: `templateId`, `studentId`, `sessionId`, `displayStepId` (FK → Step — под какой карточкой показывать; может отличаться от `template.stepId` для cross-step), `assignedById`, `createdAt`; индексы `[sessionId, displayStepId]`, `[studentId]`
   - `ExtraAssignmentCompletion` — оценка: `studentExtraAssignmentId` (unique), `grade` (Int 1|3|5), `note?`, `createdAt`
   - Добавить обратные relation на `User`, `Student`, `Session`, `Step`
   - Миграция: `pnpm db:migrate -- --name add_extra_assignments`; проверить SQL — только CREATE/ADD

2. **Zod** (`extra-assignment.ts`):
   - `stepContentSchema` переиспользовать из `validations/step.ts`
   - `createExtraAssignmentSchema`, `updateExtraAssignmentSchema`, `listExtraAssignmentsQuerySchema` (authorId?, stepId?, levelId?, title?)
   - `assignExtraAssignmentSchema` (templateId, studentId, sessionId, displayStepId)
   - `gradeExtraAssignmentSchema` (grade: 1|3|5, note?)

3. **API** — `authorizeApiRequest` с `allowedRoles: ['TEACHER','MANAGER','SUPER_ADMIN']`:
   - `GET /api/extra-assignments` — список всех шаблонов с `author.name`, `step.order/title/level`; фильтры из query
   - `POST` — создать шаблон; `authorId = session.user.id`
   - `PATCH/DELETE /api/extra-assignments/[id]` — PATCH любому с ролью; DELETE только если `authorId === session.user.id` (иначе 403)
   - `POST /api/extra-assignments/assign` — `authorizeApiRequest` с `context: { studentId }`; создать `StudentExtraAssignment`; **не** вызывать `recalculateStudentStepIdx`
   - `PATCH .../instances/[id]/grade` — upsert `ExtraAssignmentCompletion`; проверить доступ к ученику через тот же authorize
   - `GET` instances: добавить query endpoint (например `GET /api/extra-assignments/instances?studentId&date` или расширить существующий sessions GET) — вернуть instances с template (title, content, author), completion, displayStepId для урока

4. **Entities** — React Query hooks:
   - `useExtraAssignments(filters)` → GET catalog
   - `useSessionExtraAssignments(studentId, date)` → GET instances
   - мутации: `useCreateExtraAssignment`, `useUpdateExtraAssignment`, `useDeleteExtraAssignment`, `useAssignExtraAssignment`, `useGradeExtraAssignment` с `invalidateQueries` по ключам `['extra-assignments']`, `['session-extra-assignments', studentId, date]`

**Verify:**
```bash
pnpm exec prisma generate
pnpm exec tsc --noEmit
```
Проверить migration.sql: нет DROP/DELETE TABLE.

**Done:** Схема применена локально; API CRUD шаблонов, assign и grade отвечают JSON envelope; hooks типизированы.
