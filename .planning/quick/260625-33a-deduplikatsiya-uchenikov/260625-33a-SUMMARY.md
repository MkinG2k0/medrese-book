# Quick Task 260625-33a — Summary

**Task:** дублирование данных и ученики в «Сменить учётку»

## Root cause

`RememberedAccountsSelect` показывает записи из `localStorage` (`medrese-remembered-accounts`):

1. **Ученики в списке** — `shouldRememberAccount` разрешал все роли кроме SUPER_ADMIN/MANAGER, включая STUDENT. При входе ученика на общем компьютере его код попадал в список «Сменить учётку» у учителя/менеджера.
2. **Дубли «Учитель Ахмад»** — дедупликация шла по `id`, а не по `code`. После reseed БД один и тот же код мог сохраниться дважды с разными id.

## Fix

`src/features/auth/lib/remembered-accounts-storage.ts`:

- `shouldRememberAccount` — только `TEACHER`
- `normalizeRememberedAccounts` — фильтр ролей + дедуп по `code`
- `getRememberedAccounts` — при чтении перезаписывает localStorage очищенным списком (убирает старых учеников и дубли без ручной очистки)
