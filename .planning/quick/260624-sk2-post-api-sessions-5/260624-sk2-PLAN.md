# Quick Plan 260624-sk2: Оптимизировать POST /api/sessions (5 сек)

## Проблема

POST `/api/sessions` ~5 с из-за цепочки последовательных запросов к удалённой БД:
1. Дублирующий `student.findUnique` после `authorizeApiRequest` (уже проверил ученика)
2. `recalculateStudentStepIdx` загружает **все** completions ученика через `include`
3. `getStepOffsetForLevel` каждый раз делает `COUNT` по всей таблице steps

## Задачи

### Task 1: Оптимизировать recalculateStudentStepIdx
- Загружать только completions текущего уровня (`stepId in levelStepIds`)
- Использовать `select` вместо полного include
- `orderBy: createdAt asc` для согласованности с journal-actions

### Task 2: Кэшировать offsets уровней
- `getStepOffsetForLevel` → использовать `getLevelStepOffsets()` с module-level cache
- Экспорт `invalidateStepOffsetCache()` для будущих мутаций программы

### Task 3: Упростить POST /api/sessions
- Убрать дублирующий запрос student (auth уже проверил)
- Обернуть save + recalculate в `$transaction`

## Verify
- TypeScript компилируется
- Логика recalculate не меняется (только объём данных)
