# Электронный дневник медресе

## What This Is

Веб-приложение для медресе: ведение журнала посещаемости и успеваемости учеников по учебным предметам. Учителя проводят уроки и выставляют оценки, менеджеры управляют предметами, программами и персоналом, ученики видят прогресс по каждому предмету.

Текущий этап — milestone v2.0 «Система предметов»: переход от одной глобальной программы Корана к мультипредметной модели с независимым прогрессом, журналом и аналитикой по предмету.

## Core Value

Учитель и менеджер видят реальный прогресс каждого ученика — что пройдено, сколько времени заняло обучение, где отстаёт — и могут вовремя вмешаться (допзадания, смена преподавателя, контроль нормативов).

## Current Milestone: v2.0 Система предметов

**Goal:** Перейти к мультипредметной модели — прогресс, журнал и аналитика ведутся по предмету; программа — шаблон на предмет.

**Target features:**
- CRUD предметов (MANAGER / SUPER_ADMIN)
- Учебная программа предмета: уровни → шаги (как сейчас, но на каждый предмет)
- Группа привязана к одному предмету; ученик может быть в нескольких группах
- Независимый прогресс ученика по каждому предмету
- Сессия: ученик + дата + предмет (посещаемость и оценки)
- Журнал: выбор группы → прогресс по предмету группы
- Аналитика: селект предмета
- Новая модель данных (fresh start, без миграции старых completions)

### Out of Scope

- Мобильное нативное приложение — веб достаточен для медресе
- Чат в v1 — отложен до фазы 3; в первых релизах только событийные уведомления
- Real-time WebSocket-чат в MVP — начнём с асинхронной модели при внедрении чата
- S3-хранилище файлов — локальные загрузки в `public/uploads/` достаточны
- Полная переработка дизайна — точечные UI-улучшения по бэклогу

## Context

**Существующая кодовая база** (см. `.planning/codebase/`):
- Next.js 16, React 19, Ant Design, Tailwind, Prisma 7, PostgreSQL (Neon)
- Журнал: server actions + REST API + React Query + Zustand для черновика урока
- Глобальный `currentStepIdx` через offset уровней — зона техдолга
- API-авторизация разрозненная (middleware не покрывает `/api/*`)
- `next-auth@5.0.0-beta.30`, 6-значные коды без rate-limit

**Бэклог:** утверждён пользователем 2026-06-24 с приоритетами MVP → средний → низкий.

**Решения по скоупу v1:**
- Высокий приоритет — поэтапно, не одним релизом
- Фаза 1: только аналитика ученика (без аналитики преподавателя)
- Длительность урока: ручной «Начать» / «Завершить»
- Статусы ученика — фаза 2

## Constraints

- **Tech stack**: сохранить Next.js 16 + Prisma + PostgreSQL + FSD — миграция нецелесообразна
- **Architecture**: новые фичи в `src/features/`, следовать существующим паттернам server actions / API routes
- **Language**: UI и документация на русском
- **Roles**: существующая модель ролей; супер-админ вне чата
- **Data**: не ломать существующие сессии и completions; миграции через Prisma
- **Security**: закрыть известные уязвимости API (см. CONCERNS.md) в рамках фазы безопасности
