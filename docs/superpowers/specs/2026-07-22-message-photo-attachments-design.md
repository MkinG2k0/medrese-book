# Дизайн: фото во вложениях сообщений

**Дата:** 2026-07-22  
**Статус:** согласовано (подход A)

## Цель

В мессенджере можно прикреплять до 5 фото к одному сообщению. Текст не обязателен: достаточно хотя бы одного фото. Пустое сообщение (нет текста и нет фото) запрещено.

## Контекст

- Сейчас `Message` хранит только `body: String`.
- Загрузка файлов уже есть (`uploadFile` → S3 или `public/uploads`), но `/api/uploads` доступен только `SUPER_ADMIN` / `MANAGER`.
- У постов уже есть `PostMedia` + галерея — ориентир по паттерну.

## Решение

### 1. Данные

Новая модель (аналог `PostMedia`, только IMAGE):

```prisma
model MessageMedia {
  id        String  @id @default(cuid())
  messageId String
  message   Message @relation(fields: [messageId], references: [id], onDelete: Cascade)
  url       String
  sortOrder Int     @default(0)

  @@index([messageId, sortOrder])
}
```

- `Message.body` остаётся `String`, для photo-only сохраняем `""`.
- Миграция Prisma: только `CREATE TABLE` + relation на `Message`.

### 2. Валидация

`sendMessageSchema`:

- `body`: trim, max 4000, **может быть пустым**.
- `imageUrls`: массив строк-URL, max 5.
- `.refine`: `body.length > 0 || imageUrls.length > 0`.
- На сервере: принимать только URL из нашего storage (`/uploads/…` или публичный S3 prefix), MIME на этапе upload: jpeg/png/webp, разумный лимит размера (как у текущих uploads / постов).

### 3. API

| Эндпоинт | Изменение |
|----------|-----------|
| `POST /api/uploads` | Разрешить роли, которые пишут в чат: `TEACHER`, `MANAGER`, `STUDENT` (+ существующие). Ограничить типы: только изображения для messaging (или общий whitelist image). |
| `GET/POST …/messages` | DTO: `{ …, media: { id, url, sortOrder }[] }`. POST создаёт `Message` + `MessageMedia` в одной транзакции. |
| Список диалогов | Превью: если `body` пустой и есть media → «📷 Фото» / «📷 N фото». |

Уведомление `MESSAGE_RECEIVED`: если body пустой — превью «Фото» (или «N фото»).

### 4. Клиент

- `ChatPanel`: кнопка прикрепления, выбор нескольких файлов (до 5 суммарно), превью с удалением до отправки.
- Upload каждого файла → URL → `useSendMessage({ body, imageUrls })`.
- В пузыре: сетка превью (переиспользовать идею `PostMediaGallery`, упрощённо для чата) + клик → полноразмерный просмотр (Modal / Image preview antd).
- Кнопка «Отправить» активна, если есть текст **или** хотя бы одно фото (и нет незавершённого upload).

### 5. Вне скоупа

- Видео, PDF, голосовые.
- Редактирование/удаление уже отправленных вложений отдельно от сообщения.
- Сжатие на клиенте (можно позже).

## Критерии приёмки

1. Участник диалога прикрепляет 1–5 фото и отправляет с текстом или без.
2. Получатель видит фото в истории; клик открывает крупный вид.
3. В списке диалогов и в уведомлении photo-only отображается как «Фото», а не пустая строка.
4. Сообщение без текста и без фото отклоняется валидацией.
5. Ученик/учитель могут загружать фото для чата (не только менеджер).
