# Бухгалтерия — руководство для бухгалтера

> **Версия документа:** июль 2026  
> Описывает модуль «Бухгалтерия» в текущем виде. При доработках системы документ нужно обновлять.

---

## 1. Назначение

Модуль «Бухгалтерия» в дневнике медресе нужен, чтобы:

- учитывать **оплату обучения** учениками;
- начислять и **выплачивать зарплату** учителям (по фактическим часам из журнала);
- фиксировать **расходы** и **пожертвования**;
- видеть **общий финансовый баланс** школы;
- **закрывать месяцы**, чтобы нельзя было менять прошлые периоды задним числом.

Это **не** полноценная бухгалтерская программа с двойной записью (дебет/кредит). Это **журнал операций**: каждая сумма — отдельная запись, без связи «платёж ученика → зарплата учителю».

---

## 2. Кто видит модуль

| Роль | Доступ |
|------|--------|
| **Бухгалтер** | Полный доступ ко всем разделам «Бухгалтерия» |
| **Учитель** | Только «Моя зарплата» — свои часы и начисления, без данных других и без баланса школы |
| Менеджер, ученик, админ | Модуль **не видят** |

Учётная запись бухгалтера создаёт менеджер: **Пользователи → Создать пользователя → роль «Бухгалтер»**.

---

## 3. Вход в систему

1. Откройте сайт дневника.
2. Введите **6-значный код доступа**, выданный администратором.
3. После входа откроется раздел **«Бухгалтерия»** (дашборд).

В боковом меню доступны пункты:

- **Бухгалтерия** — сводка и закрытие месяца  
- **Платежи** — оплаты учеников  
- **Зарплаты** — часы и выплаты учителям  
- **Расходы** — расходы и пожертвования  
- **Операции** — единый журнал всех событий  

---

## 4. Деньги в системе

- Все суммы хранятся **в копейках** (целые числа), в интерфейсе показываются **в рублях** (₽).
- **Тариф ученика по умолчанию** — 2 000 ₽ в месяц (задаётся при создании ученика в системе; для новых учеников подставляется автоматически).
- Способы оплаты: **наличные**, **карта**, **перевод**.

---

## 5. Дашборд («Бухгалтерия»)

Вверху — **выбор месяца** (календарь «месяц/год»). Все карточки на дашборде относятся к **выбранному месяцу**, кроме общего баланса.

### Карточки

| Показатель | Что означает |
|------------|--------------|
| **Баланс школы** | Все поступления (платежи учеников + пожертвования) минус все расходы и выплаты зарплат **за всё время** |
| **Обязательства (ЗП)** | Зарплата **подтверждена**, но **ещё не выплачена** полностью |
| **Поступило за месяц** | Сумма платежей учеников с датой в выбранном месяце |
| **Начислено за месяц** | Сумма начислений обучения (тарифы) за выбранный месяц |
| **Сумма долгов** | Общий долг всех **активных** учеников (см. ниже) |
| **Должников** | Число активных учеников с отрицательным сальдо |
| **Наличные / Карта / Перевод** | Остаток по каждому способу оплаты (приходы минус расходы и выплаты с тем же способом) |

### Кнопки

**«Начислить за месяц»**  
Создаёт **начисление обучения** каждому **активному** ученику на выбранный месяц в размере его тарифа.  
Повторное нажатие **не создаёт дубликаты** — безопасно нажимать снова.  
То же самое может выполняться **автоматически 1-го числа** (настраивает администратор сервера).

**«Закрыть месяц»**  
Блокирует **любые новые операции с датой внутри этого месяца** (платежи, расходы, выплаты и т.д.).  
Отменить закрытие через интерфейс **нельзя** — только обращение к администратору системы.

---

## 6. Платежи учеников

Таблица по **выбранному месяцу**:

| Колонка | Смысл |
|---------|--------|
| Ученик, группа | Кто платил |
| Тариф | Месячная сумма обучения |
| Оплачено | Сколько внесено **в этом месяце** (по дате платежа) |
| Сальдо | **Все платежи минус все начисления** за всё время (не только за месяц) |
| Статус | См. таблицу статусов ниже |

**Фильтр «Только должники»** — показывает учеников с отрицательным сальдо.

**«+ Платёж»** — новая оплата: ученик, сумма (в рублях), способ, комментарий.  
Дата платежа подставляется **сегодняшняя**.

### Статусы ученика

| Статус | Значение |
|--------|----------|
| **Оплачено** | За выбранный месяц внесено не меньше тарифа, общего долга нет |
| **Частично** | За месяц внесена часть тарифа |
| **Долг N мес.** | Общее сальдо отрицательное (ученик должен школе) |
| **Аванс** | Платежей больше, чем начислений — переплата |

### Сальдо ученика

- **Отрицательное** — долг школе.  
- **Положительное** — аванс (переплата).  
- **Ноль** — всё сведено.

Начисления создаются кнопкой или автоматически; **вручную начисление в таблице не редактируется**.

---

## 7. Зарплаты

Таблица по **выбранному месяцу** для каждого учителя:

| Колонка | Смысл |
|---------|--------|
| Часы | Сумма длительности **завершённых** уроков из журнала (клик — список уроков) |
| Ставка | Почасовая ставка (если не задана — **0 ₽**, начисление будет 0) |
| Начислено | Часы × ставка (пересчитывается при открытии страницы) |
| Статус | Черновик → Подтверждено → Выплачено |

### Статусы начисления

1. **Черновик** — расчёт есть, часы ещё не утверждены.  
2. **Подтверждено** — бухгалтер проверил часы; можно выплачивать.  
3. **Выплачено** — выплата проведена (полностью или система отметила после выплаты).

### Действия

- **«Подтвердить»** — перевести черновик в «Подтверждено».  
  **Нельзя**, если есть уроки с пометкой **«Проверить»** (урок длиннее 3 часов — учитель, вероятно, забыл нажать «Закончить урок»). Сначала разберитесь с такими уроками (см. раздел 11).
- **«Выплата»** — ручная выплата: сумма и способ. Автоматических выплат **нет**.

### Откуда берутся часы

Только из **журнала учителя**: время между «Начать урок» и «Закончить урок».  
Ручной ввод часов как основной способ **не предусмотрен**.

---

## 8. Расходы и пожертвования

Две вкладки:

### Расходы

Категории: **расходники**, **коммунальные**, **аренда**, **прочее**.  
Поля: дата, сумма, способ оплаты, комментарий.

### Пожертвования (садака)

Поля: дата, сумма, способ, комментарий.

У каждой записи может быть кнопка **«Сторнировать»** (см. раздел 10).

---

## 9. Журнал операций

Единая лента **всех финансовых событий**:

- платежи учеников;
- расходы и пожертвования;
- выплаты зарплат;
- начисления обучения и начисления зарплаты.

Можно отфильтровать **период** и **тип** операции.  
Записи **нельзя редактировать и удалять** из этого экрана — только просмотр.

---

## 10. Главные правила (обязательно к прочтению)

### 10.1. Запись нельзя изменить и удалить

Любая финансовая запись после сохранения **не редактируется и не удаляется**.  
Ошибку исправляют через **сторнирование**.

### 10.2. Сторнирование

- В интерфейсе кнопка называется **«Сторнировать»**, не «Удалить».
- Создаётся **новая запись** с **отрицательной суммой** и **обязательным комментарием** (причина).
- Исходная запись остаётся в журнале.

**Сейчас в интерфейсе:** сторно доступно для **расходов** и **пожертвований**.  
Для платежей учеников и выплат зарплаты сторно реализовано в системе, но **отдельной кнопки в таблице платежей пока нет** — при необходимости обратитесь к администратору.

### 10.3. Закрытие месяца

После **«Закрыть месяц»** для выбранного месяца (например, `2026-07`):

- нельзя провести **новую** операцию с **датой в этом месяце**;
- появится сообщение: *«Месяц 2026-07 закрыт для операций»*.

Корректировки после закрытия — **только записями в текущем открытом месяце** или через сторно (где доступно).

**Важно:** новые платежи и расходы получают **сегодняшнюю дату**. Если сегодня уже другой месяц, а прошлый закрыт — провести «задним числом» в закрытый месяц **нельзя**. Это сделано намеренно.

### 10.4. Разделение денег

Платежи учеников **не привязаны** к зарплатам учителей.  
Деньги в системе **обезличены**: есть общий баланс школы, но нет «перевода с ученика на учителя».

### 10.5. Кто попадает в начисления обучения

Только ученики со статусом **«Активен»**.  
На паузе или в архиве — **не начисляются**.

---

## 11. Уроки «Проверить» (аномалия > 3 часов)

Если урок в журнале длился **больше 3 часов**, в списке уроков учителя он помечается **«Проверить»**.

**До подтверждения зарплаты** за месяц все такие уроки должны быть разобраны:

- уточнить у учителя (забыл завершить урок?);
- при необходимости скорректировать длительность (в текущей версии корректировка через интерфейс **в разработке**; временно — через администратора).

Пока есть неразрешённые аномалии, кнопка **«Подтвердить»** для зарплаты **не сработает**.

---

## 12. Что видит учитель

В меню учителя есть пункт **«Моя зарплата»**:

- часы за выбранный месяц;
- начисленная сумма;
- сколько уже выплачено;
- статус (черновик / подтверждено / выплачено).

Учитель **не видит**: баланс школы, платежи других, расходы, зарплаты коллег.

---

## 13. Автоматическое начисление обучения (1-е число)

Администратор может настроить **автоматический запуск 1-го числа каждого месяца**: система создаёт начисления всем активным ученикам по их тарифу (то же, что кнопка «Начислить за месяц»).

Бухгалтеру достаточно **проверить** дашборд после 1-го числа и при необходимости нажать «Начислить за месяц» вручную (повтор безопасен).

---

## 14. Типичный месяц (чек-лист)

### Начало месяца

1. Убедиться, что **прошлый месяц закрыт** (если ещё нет — закрыть после сверки).  
2. Дождаться или нажать **«Начислить за месяц»** для нового месяца.  
3. Принимать **платежи** учеников по мере поступления.  
4. Фиксировать **расходы** и **пожертвования**.

### Середина / конец месяца — зарплаты

1. Открыть **«Зарплаты»**, выбрать месяц.  
2. Проверить **часы** (клик по часам → список уроков).  
3. Убрать пометки **«Проверить»** (согласовать с учителями / админом).  
4. Нажать **«Подтвердить»** по каждому учителю.  
5. Провести **«Выплату»** (ручно, когда деньги реально выданы).

### Конец месяца

1. Сверить **дашборд**, **платежи**, **журнал операций**.  
2. Нажать **«Закрыть месяц»**.  
3. После закрытия операции с датами этого месяца **заблокированы**.

## 15. Ограничения текущей версии

Зафиксировано на момент июля 2026 — **может измениться**:

| Функция | Статус |
|---------|--------|
| Дашборд, платежи, зарплаты, расходы, журнал | Работает |
| Закрытие месяца, начисление обучения | Работает |
| Сторно расходов и пожертвований | Работает в интерфейсе |
| Сторно платежей и выплат зарплаты | В системе есть, **кнопки в UI платежей пока нет** |
| Редактирование тарифа ученика | **Нет отдельного экрана** в бухгалтерии (тариф при создании ученика / в БД) |
| Установка почасовой ставки учителя | **Нет экрана** — ставки задаёт администратор |
| Корректировка длительности урока | **Нет кнопки в UI** — через администратора |
| Экспорт в Excel | Реализован на сервере, **кнопок в интерфейсе пока нет** |
| Уведомления для бухгалтера | Нет (колокольчик скрыт) |
| Сообщения / чат | Нет доступа у роли бухгалтер |

## 16. Частые сообщения об ошибках

| Сообщение | Что делать |
|-----------|------------|
| *Месяц 2026-07 закрыт для операций* | Месяц закрыт. Новую операцию с датой в июле провести нельзя. Используйте текущий месяц или сторно (если доступно). |
| *Есть N урок(ов) с аномальной длительностью* | Разберите уроки > 3 ч, затем снова «Подтвердить». |
| *Можно подтвердить только черновик* | Зарплата уже подтверждена или выплачена. |
| *Сначала подтвердите начисление* | Нельзя выплатить, пока не нажато «Подтвердить». |
| *Запись уже сторнирована* | Повторное сторно одной операции невозможно. |
| *Комментарий обязателен для сторно* | Укажите причину в диалоге сторнирования. |

---

## 17. Журнал действий (аудит)

Все значимые операции (платежи, расходы, закрытие месяца, сторно и т.д.) записываются в **«Журнал действий»** (доступен менеджеру/админу): кто, когда и что сделал.  
Бухгалтер этот раздел **не видит**, но его действия там фиксируются.

---

## 18. Контакты и поддержка

При сбоях, необходимости **открыть закрытый месяц**, задать **ставку учителю** или **тариф ученика** обращайтесь к **администратору системы** (менеджер медресе / техническая поддержка).
